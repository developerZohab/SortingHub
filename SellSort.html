üéØ Real World Scenario
Sorting loose files by color before sorting them alphabetically. Shell sort organizes items that are "far apart" first (big steps), then refines the sort with smaller steps. This pre-sorting makes the final cleanup much faster than standard insertion sort.
‚öôÔ∏è How It Works
  1. Start with a large Gap (e.g., n/2).
  2. Compare elements that are gap distance apart.
  3. If they are out of order, Swap them.
  4. Reduce the Gap (e.g., gap / 2) and repeat.
  5. The final pass has a Gap of 1, effectively performing a quick Insertion Sort on a nearly sorted list.

Metric Complexity Explanation
Worst Case O(n¬≤) Depends heavily on the Gap Sequence. Using simple n/2 gaps can result in worst-case quadratic time, but better sequences exist.
Average Case O(n log n) Or often cited as O(n^1.25) or O(n^1.5). It is significantly faster than simple O(n^2) sorts for medium-sized arrays.
