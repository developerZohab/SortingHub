üéØ Real World Scenario
Organizing a date-based file archive: First, you sort all files by Day (1-31). Then, keeping that order, you sort them by Month (Jan-Dec). Finally, you sort by Year. By the end, the files are perfectly chronological. This "Least Significant Digit" approach keeps prior sorting intact.
‚öôÔ∏è How It Works (LSD)
  1. Find the maximum number to know the max digits (e.g., hundreds place).
  2. Pass 1 (Ones): Group numbers into buckets 0-9 based on the last digit.
  3. Collect: Put them back into the array in bucket order (0 to 9).
  4. Pass 2 (Tens): Group by the second digit. Collect again.
  5. Repeat until the most significant digit is processed.

üìä Elaborated Complexity Analysis
Metric Complexity Explanation
Best Case O(nk) Performance is linear relative to input size (n) and number of digits (k). It processes every digit regardless of order.
Average Case O(nk) Consistent performance. If k is small (e.g., 32-bit integers), it is effectively linear O(n).
Worst Case O(nk) Occurs when all numbers have the maximum number of digits possible for the data type.
Space O(n + k) Not In-Place. Requires buckets to store all n elements, plus counting arrays of size k (base 10).
