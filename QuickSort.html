üéØ Real World Scenario
Sorting a class by height: You pick one student (the Pivot) and ask everyone shorter to go to the left wall and everyone taller to the right wall. You then repeat this process for the left group and the right group.
‚öôÔ∏è How It Works (Divide & Conquer)
  1. Choose a Pivot element (usually the last element).
  2. Partition the array: Move elements smaller than Pivot to the left, larger to the right.
  3. Place the Pivot in its correct sorted position (between left and right parts).
  4. Recursively apply the same steps to the Left Sub-array and Right Sub-array.
  5. Base Case: If a sub-array has 0 or 1 elements, it is sorted.

Metric Complexity Explanation
Worst Case O(n¬≤) Occurs when the pivot is always the smallest or largest element (e.g., sorted array with last-element pivot), creating unbalanced partitions.
Best Case O(n log n) Occurs when the pivot partitions the array into two roughly equal halves every time.
Average Case O(n log n) On random data, partitions are usually balanced enough to achieve log-linear time. It is generally faster than Merge Sort due to cache locality.
Space O(log n) Stack Space. Although it sorts in-place, the recursion stack takes logarithmic space (balanced) or linear space (worst case).
