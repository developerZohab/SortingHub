üéØ Real World Scenario
Counting votes in an election: You have boxes for each candidate. As you pick up a ballot, you just drop it into the candidate's box. Once all ballots are distributed, you just collect the votes from Candidate A's box, then B's, and so on. No comparisons needed!
‚öôÔ∏è How It Works
  1. Determine the range of the input (e.g., 0 to 9).
  2. Create "buckets" or count arrays for each possible value.
  3. Count: Iterate through the input and place each number into its corresponding bucket.
  4. Collect: Reconstruct the array by gathering elements from bucket 0, then 1, up to the max value.

üìä Elaborated Complexity Analysis
Metric Complexity Explanation
Best Case O(n + k) Where n is input size and k is the range of values (max - min). Fast when k is small.
Average Case O(n + k) Performance is linear regardless of the data order. It does not rely on comparisons.
Worst Case O(n + k) Even if the array is reverse sorted, it still takes the same time. Note: if k is very large (e.g., n^2), it becomes O(n^2).
Space O(k) Not In-Place. Requires an auxiliary Count Array of size k to store frequencies.
