üéØ Real World Scenario
Organizing books on a shelf: Compare adjacent books, swap if out of order, repeat until sorted.
‚öôÔ∏è How It Works
  1. Start from the first element of the array.
  2. Compare the current element with the one immediately next to it.
  3. Swap them if they are in the wrong order (i.e., current > next).
  4. Move to the next pair and repeat until the end of the array.
  5. After one full pass, the largest element "bubbles" to the correct position at the end.
  6. Repeat the process for the remaining elements until sorted.

Metric Complexity Explanation
Worst Case O(n¬≤) Occurs when the array is sorted in reverse order. The outer loop runs n times, and for each iteration, the inner loop runs n-i times.
Best Case O(n) Occurs when the array is already sorted. With an optimization flag (swapped = false), it terminates early.
Average Case O(n¬≤) Occurs on random data. On average, half the elements are out of order, still requiring nested loops roughly proportional to n¬≤.
Space O(1) In-Place Sort. No additional data structures are created. Swapping happens directly within the original array using a temp var.
